# Vite Dependency Resolution Report

## Executive Summary
On March 14, 2026, the Vercel build process failed due to a dependency resolution conflict involving `vite`, `@vitejs/plugin-react-swc`, and `lovable-tagger`. This report documents the analysis and implementation of the fix, which involved downgrading Vite from version 8.0.0 back to 7.3.1 to maintain compatibility with the existing ecosystem.

## Issue Description
Deployments to Vercel were failing during the `npm install` phase with the following error:

```text
npm error Conflicting peer dependency: vite@7.3.1
npm error node_modules/vite
npm error   peer vite@"^4 || ^5 || ^6 || ^7" from @vitejs/plugin-react-swc@3.11.0
npm error   node_modules/@vitejs/plugin-react-swc
npm error     dev @vitejs/plugin-react-swc@"^3.11.0" from the root project
```

The build exited with error code 1, preventing the production site from updating.

## Root Cause Analysis
The investigation revealed that the project was attempting to use `vite@8.0.0`, which was recently released. This created several conflicts:

1.  **lovable-tagger**: This critical development tool has a strict peer dependency requirement for Vite version less than 8 (`>=5.0.0 <8.0.0`).
2.  **@vitejs/plugin-react-swc**: The installed version (3.11.0) was configured to support Vite versions 4 through 7.

While `npm` could find a way to resolve some dependencies, the presence of these strict peer requirements caused a complete failure when attempting to build in a clean Vercel environment.

## Changes Implemented

### 1. Package Configuration
The `package.json` was updated to restore compatibility:
- **Vite**: Downgraded from `^8.0.0` to `^7.3.1`.
- **@vitejs/plugin-react-swc**: Updated from `^3.11.0` to `^4.3.0` (to ensure the best available support for the Vite 7 branch).

### 2. Lockfile Update
A fresh `npm install` was performed locally to regenerate the `package-lock.json`, ensuring that all sub-dependencies are correctly pinned to versions that support Vite 7.
